1. Introduction:

  • 1. The system can take in addresses, either individually through text boxes or in a bulk list, and normalize them. A successful system operation will result in a list of normalized addresses and a list of invalid addresses that were unable to be normalized.

2. Glossary:

  • 1. Valid address: address does not have all possible information but can still be normalized.
  • 2. Invalid address: Address does not have enough information to determine what the complete address should be.
  • 3. Normalized Address: Address fits the format requirements as defined by the United States Postal Service.
  • 4. Locally: Within the system where user began the process.

3. Functional Requirements:

  • 1. The system shall be able to normalize mailing addresses given the mailing addresses are valid (questions 3.2, 3.3)
  • 2. The system shall not be able to normalize mailing addresses given the mailing addresses are invalid (questions 3.2, 3.3)
  • 3. The user shall be able to normalize a single entry (question 5.2)
  • 4. The user shall be able to normalize a file (e.g. .txt and .csv) with bulk entries (question 5.2)
  • 5. The system shall be able to distinguish between valid and invalid entries and output them into separate files (question 5.3)

4. Non-Functional Requirements:

  • 1. The system must be a desktop application (question 4.1.a)
  • 2. The system must perform all operations on the data locally (question 4.4)
  • 3. The system must be able to perform all operations offline (question 4.5)
  • 4. In the case of an unexpected program termination, any in-progress normalization from before the termination shall not be retrievable (question 3.7)
  • 5. The system must have a graphical user interface (question 4.1.a)
